# Labeling Guide

## Goal
이 프로젝트의 목표는 한국어 리뷰를 아래 두 범주로 분류하는 것이다.

- 진심 리뷰 (`genuine`)
- 좋아요성 리뷰 (`promotion`)

이 문서는 사람이 직접 라벨링할 때 기준을 통일하기 위한 가이드다.

## Core Principle
라벨은 플랫폼이 아니라 "리뷰 내용"을 기준으로 판단한다.

다음 정보는 참고는 가능하지만 정답 라벨의 직접 기준으로 쓰지 않는다.

- 플랫폼 종류
- 리뷰이벤트 여부
- 재주문 횟수
- 별점
- 사진 유무

이런 메타데이터는 나중에 모델 feature 후보로 사용할 수 있다.

## Labels

### `genuine`
아래에 해당하면 진심 리뷰 (`genuine`)로 본다.

- 상품, 메뉴, 맛, 양, 배송, 포장, 사용감, 품질 등 구체 정보가 있다.
- 실제 경험을 바탕으로 쓴 내용이 중심이다.
- 짧더라도 특정한 경험이나 평가 포인트가 드러난다.

예시
- "양이 생각보다 많고 국물이 진했어요. 다만 면은 조금 불었습니다."
- "포장은 깔끔했고 배송은 35분 정도 걸렸어요. 떡볶이는 달달한 편입니다."
- "리뷰이벤트로 받았지만 치킨은 바삭했고 소스는 조금 짰어요."

### `promotion`
아래에 해당하면 좋아요성 리뷰 (`promotion`)로 본다.

- 이벤트 참여 문구가 핵심이다.
- 구체 경험 정보 없이 칭찬이나 감사 표현만 있다.
- 지나치게 짧거나 템플릿성 표현이 많다.
- 길어도 템플릿처럼 반복되고 정보 밀도가 낮으면 `promotion` 가능하다.
- 문장 수는 많아도 상품 설명, 감탄사, 과장된 추천 문구가 대부분이면 `promotion`로 본다.
- 실제로 먹었거나 사용한 흔적이 거의 없다.

예시
- "리뷰이벤트 참여합니다. 잘 먹었습니다."
- "서비스 감사합니다. 맛있어요. 재주문할게요."
- "사장님 친절하세요. 최고입니다."

### `uncertain`
판단이 애매하면 제외 후보로 둔다.

- 실제 경험은 있는데 정보량이 너무 적다.
- 이벤트 문구와 실제 경험이 섞여 있는데 어느 쪽이 중심인지 애매하다.
- 문장이 너무 짧거나 비문이라 의도 파악이 어렵다.
- 복사/반복 리뷰처럼 보이나 확신이 없다.

예시
- "맛있어요 또 시킬게요"
- "좋았습니다"
- "이벤트로 작성하지만 괜찮았어요"

## Detailed Decision Rules
라벨링할 때는 아래 순서로 판단한다.

1. 구체 경험 정보가 있는가?
- 메뉴명, 배송 시간, 포장 상태, 맛, 양, 식감, 품질, 사용 상황 등이 보이면 `genuine` 쪽으로 본다.

2. 리뷰의 중심이 이벤트 참여 문구인가?
- 이벤트 참여 사실만 드러나고 구체 경험이 없으면 `promotion`로 본다.

3. 칭찬이 있어도 근거가 있는가?
- "맛있어요"만 있으면 약하다.
- "매콤하고 떡이 쫄깃해서 맛있어요"면 `genuine`다.

4. 짧더라도 구체성이 있으면 `genuine` 가능
- 길이보다 정보 밀도가 더 중요하다.

5. 길다고 해서 자동으로 `genuine`로 보지 않는다
- 매우 긴 리뷰라도 비슷한 칭찬 반복, 과장된 추천, 상품소개식 문장 위주면 `promotion` 가능하다.

6. 애매하면 `uncertain`
- 억지로 2분류에 넣지 않는다.

## Feature Leakage Warning
라벨링 중 아래 정보가 너무 강하게 정답을 암시하면 메모해둔다.

- "리뷰이벤트" 같은 고정 문구
- 플랫폼 고유 UI 문구가 그대로 복사된 경우
- 재주문 횟수만 보고 진정성을 단정하는 경우

이런 정보는 모델이 쉽게 외울 수 있으므로 나중에 실험에서 분리 검토한다.

## Recommended Labeling Workflow
1. 샘플을 읽는다.
2. 텍스트만 먼저 보고 1차 판단한다.
3. 필요할 때만 메타데이터를 참고한다.
4. `genuine`, `promotion`, `uncertain` 중 하나를 기록한다.
5. 판단 이유를 짧게 메모한다.

## Recommended Note Format
- `specific_taste`
- `specific_delivery`
- `event_phrase_only`
- `too_short`
- `long_but_generic`
- `template_like_long_review`
- `generic_praise`
- `mixed_signal`
- `unclear_text`

## Quality Control
- 처음 50개는 라벨링 후 다시 재검토한다.
- 기준이 흔들리면 이 문서를 먼저 수정한다.
- 가능하면 2회차 검토에서 `uncertain` 일부를 재분류한다.

## Important Reminder
이 프로젝트의 목표는 "이벤트 리뷰 문구 검출기"를 만드는 것이 아니다.

목표는 "형식적이고 정보가 빈약한 리뷰"와 "실제 경험 정보가 있는 리뷰"를 구분하는 것이다.
